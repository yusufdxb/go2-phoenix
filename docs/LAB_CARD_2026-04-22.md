## Block 3 ŌĆö Parquet collection for loop closure (2ŌĆō3 hr)

**Goal:** 5ŌĆō10 parquets with Ōēź4 distinct failure modes. **Diversity > quantity.**

This is the Gate-9-enabling block. Everything else is ceremony; this is the only work on Day 1 that directly unlocks the loop closure.

### Failure mode menu (pick 4+)

| Mode | Perturbation | Expected detector label |
|---|---|---|
| Slip | Towel / plastic sheet under one foot | `slip` |
| Push recovery | Lateral shove from operator | `attitude_deviation` or recovery |
| Ramp descent | 10┬░ ramp, walk down | `lean_forward` / `pitch` |
| Ramp ascent | Same ramp, walk up | `lean_back` |
| Terrain step | 3 cm book / block | `contact_transient` |
| Yaw disturbance | Shove robot's hindquarters | `yaw_slip` |
| Sudden stop | Step cmd_vel from 0.5 to 0 instantly | Stumble on deceleration |
| Off-gait freeze | Hold deadman mid-walk | Freeze + recover |

### Protocol per scenario

1. Arm deadman. Robot walking at `(vx=0.3, vyaw=0)`.
2. Apply perturbation.
3. **Do NOT catch the robot.** Let the failure happen. Tether arrests fall if needed.
4. Abort after failure or 10 s post-perturbation.
5. Confirm parquet written, name it.
6. Reset robot to sit. Next scenario.

**Name parquets descriptively:** `data/failures/pqa_<mode>_<ts>.parquet` where `<mode>` Ōłł {slip, push_lat, ramp_dn, ramp_up, step_up, yaw_shove, stop_sudden, deadman_freeze}. The loop_closure script filters by prefix.

### Diversity check (do this BEFORE leaving the lab)

- [ ] Parquets span Ōēź4 different `<mode>` prefixes
- [ ] At least 2 parquets per chosen mode (baseline + replicate)
- [ ] All parquets have `rows > 200` (not aborted-too-early)
- [ ] `phoenix.real_world.failure_detector` labels each one with a non-empty failure ŌĆö run a quick CLI check, not full validation

If diversity fails, extend Block 3. Don't pack up with 8 slip parquets ŌĆö that's 1 failure mode with replicates and the loop closure has nothing to learn from.
