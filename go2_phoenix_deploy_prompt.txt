4. OBSERVATION LAYOUT MISMATCH (CRITICAL — READ BEFORE LAUNCH)
----------------------------------------------------------------

The baseline policy was trained on `Isaac-Velocity-Rough-Unitree-Go2-v0` which uses a 235-dim observation:

    [ 48-dim proprioception ] + [ 187-dim height-scanner readings ]

The real GO2 DOES NOT HAVE a height scanner. The repo's `ObservationBuilder` (in `src/phoenix/sim2real/observation.py`) only builds the 48-dim proprioceptive prefix. There is a real mismatch that the on-robot deployment must resolve.

You have three options, in order of preference:

  (a) DEFAULT — stub the height scan with a 187-dim zero vector. Expected failure mode: policy will be fragile on uneven terrain, may tip at edges/steps, because it was trained to rely on a scan it now cannot see. This is acceptable for a first on-stand test and for flat-indoor rollouts.

  (b) If the Jetson has a live RGBD or 2D LiDAR stream (check `ros2 topic list | grep -Ei "depth|scan|lidar|points"`), generate a synthetic height scan by projecting the cloud onto a ground-plane grid matching Isaac Lab's raycaster pattern (1.6 m x 1.0 m, 0.1 m spacing = 17 x 11 = 187 points). This is non-trivial — only attempt it if the human explicitly asks.

  (c) REFUSE to deploy and ask the human to retrain on the flat task (`Isaac-Velocity-Flat-Unitree-Go2-v0`, obs dim = 48). This yields a policy that matches what the robot can actually observe.

If you choose (a), BEFORE launching, paste the exact observation construction snippet you will use into the terminal for the human to sanity-check. Example:

    # Height scan stub (187 zeros appended to 48-dim proprio)
    import numpy as np
    proprio = obs_builder.build(...)          # shape (48,)
    height_scan = np.zeros(187, dtype=np.float32)
    obs_full = np.concatenate([proprio, height_scan]).astype(np.float32)
    assert obs_full.shape == (235,), obs_full.shape

The current `ros2_policy_node.py` does NOT do this concatenation — it passes the 48-dim proprio directly to ONNX. You will see a shape-mismatch error from onnxruntime on first launch. You MUST patch the node to append the 187-dim stub before launch, OR confirm with the human that option (c) is preferred.

If you patch, keep the change minimal (local edit in `src/phoenix/sim2real/ros2_policy_node.py` where `self.session.run(["action"], {"obs": obs})` is called — wrap `obs` with the zero-padding). Do NOT refactor observation.py itself for this single deploy run.
